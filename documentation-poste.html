🎯 Concept et Vision

La Poste de Moulinsart est un système d'automation agentic révolutionnaire où les agents Claude communiquent entre eux par email, créant un écosystème collaboratif autonome.

✨ Innovation Principale

Remplace la communication par fichiers texte par un système de messagerie électronique complet avec notifications tmux en temps réel.

🎨 Caractéristiques Uniques

  • 📧 Communication par Email : Chaque agent a sa propre adresse email
  • 🔮 Observabilité Totale : Capture et traçage de tous les prompts Claude
  • 📮 Serveur Mail Local : SMTP complet sur port 1025
  • 🎭 Interface Style BD : Inspirée de l'univers Tintin/Hergé
  • ⚡ Temps Réel : WebSocket pour les mises à jour instantanées
  • 🤖 Multi-Agents Tmux : 4 instances Claude dans des panels séparés

  • 1️⃣ Reset complet et lancer
  • 2️⃣ Attacher/Reprendre session
  • 3️⃣ Scanner toutes les sessions
  • 4️⃣ Lancer sans reset
  • 5️⃣ Nettoyer tout
  • 6️⃣ Contrôle des agents

🚀 Lancement Direct

📌 Commandes Tmux Utiles

Action Commande
Attacher à la session tmux attach -t moulinsart-agents
Naviguer entre panels Ctrl+B puis flèches
Zoom sur un panel Ctrl+B puis Z
Détacher la session Ctrl+B puis D
Envoyer commande à Nestor tmux send-keys -t moulinsart-agents:agents.0 "message" Enter
Envoyer commande à Tintin tmux send-keys -t moulinsart-agents:agents.1 "message" Enter

💉 Système de Notifications Tmux

Les notifications sont automatiquement injectées dans les panels tmux lors de la réception d'emails.

Mécanisme: Envoi de la commande echo, puis après 0.1s, envoi de C-m pour exécution

📡 API Endpoints

Oracle API (Port 3001)

Endpoint Méthode Description
/health GET Status de santé du serveur
/api/events GET Liste des événements (limit, offset)
/api/events POST Créer un nouvel événement
/api/projects GET Liste des projets actifs
/api/agents GET Status des agents
/ws WebSocket Stream temps réel

Mail Server API (Port 1080)

Endpoint Méthode Description
/api/mailbox/{email} GET Récupérer les emails d'une boîte
/api/mailboxes GET Status de toutes les boîtes
/api/send POST Envoyer un email via API
/api/reset-all-emails DELETE Vider toutes les boîtes mail
/agents GET Liste des agents avec compteurs

Exemples d'Utilisation

# Vérifier le status
curl http://localhost:3001/health

# Récupérer les emails de Nestor
curl http://localhost:1080/api/mailbox/nestor@moulinsart.local

# Envoyer un email
curl -X POST http://localhost:1080/api/send \
  -H "Content-Type: application/json" \
  -d '{
    "from": "commandant@moulinsart.local",
    "to": "nestor@moulinsart.local",
    "subject": "Mission Urgente",
    "body": "Nouvelle mission à distribuer"
  }'

# Reset toutes les boîtes
curl -X DELETE http://localhost:1080/api/reset-all-emails

🪝 Configuration des Hooks

📝 Configuration dans ~/.claude/settings.json

{
  "UserPromptSubmit": {
    "command": "python3 ~/moulinsart/oracle-observability/hooks/oracle_prompt_hook.py"
  },
  "Stop": {
    "command": "python3 ~/moulinsart/oracle-observability/hooks/oracle_stop_hook.py"
  }
}

🔄 Flux des Hooks

  1. Capture: Le hook Python capture le prompt Claude
  2. Parsing: Extraction du projet et du contexte
  3. API Call: Envoi à l'API Oracle sur port 3001
  4. Email: Génération automatique d'un email à Nestor
  5. Notification: Injection dans le panel tmux de Nestor
  6. WebSocket: Broadcast à tous les clients connectés

⚠️ Points d'Attention

  • Les hooks utilisent actuellement le port 1026 au lieu de 1025 (à corriger)
  • Les hooks sont silencieux - pas d'output pour ne pas perturber Claude
  • Les erreurs sont loggées dans /tmp/oracle_hook.log

🚨 Dépannage et Solutions

🔴 Le serveur ne démarre pas

# Vérifier si le port est libre lsof -i :3001 lsof -i :1025 lsof -i :1080 # Tuer les processus si nécessaire kill -9 [PID] # Relancer proprement ./moulinsart-manager.sh

🔴 Les hooks ne capturent pas

# Vérifier la configuration cat ~/.claude/settings.json | grep oracle # Test manuel du hook echo "Test prompt" | python3 ~/moulinsart/oracle-observability/hooks/oracle_prompt_hook.py # Vérifier les logs tail -f /tmp/oracle_hook.log

🔴 Les notifications tmux ne s'exécutent pas

# Vérifier la session tmux ls # Test manuel d'injection tmux send-keys -t moulinsart-agents:agents.0 "echo 'Test'" && \ sleep 0.1 && \ tmux send-keys -t moulinsart-agents:agents.0 C-m

🔴 Base de données corrompue

cd ~/moulinsart/oracle-observability/data mv oracle.db oracle.db.backup # Redémarrer le serveur créera une nouvelle base

🔴 Reset complet du système

# Option 1: Via le gestionnaire ./moulinsart-manager.sh # Choisir option 1 (Reset complet et lancer) # Option 2: Manuel tmux kill-server pkill -f "claude --dangerously" pkill -f "bun.*server" rm ~/moulinsart/oracle-observability/data/oracle.db ./launch-agents-tmux.sh

🎨 Notes Techniques

💡 Architecture Décisions

  • Bun.js: Choisi pour sa rapidité avec TypeScript
  • SQLite: Base simple et suffisante pour l'observabilité
  • Vue 3: Interface réactive avec Composition API
  • WebSocket: Updates temps réel sans polling
  • SMTP Custom: Contrôle total sur le routing des emails
  • Tmux: Gestion élégante de multiples instances Claude

🔒 Sécurité

  • Système local uniquement (localhost)
  • Pas d'authentification requise (usage développement)
  • Flag --dangerously-skip-permissions pour Claude
  • À ne pas exposer sur internet
