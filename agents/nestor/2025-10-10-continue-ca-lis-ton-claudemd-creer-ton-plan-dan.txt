  🎩 RÉSUMÉ: POURQUOI API TASKS > MODE PLAN

  LA RÉPONSE SIMPLE:

  Mode Plan = Liste dans ma tête (mémoire volatile)
  API Tasks = Liste dans une base de données (mémoire persistante)

  POURQUOI JE ME PERDS AVEC LE MODE PLAN:

  Quand je fais ExitPlanMode, je crée un plan Markdown comme:
  1. Étape 1: Faire X
  2. Étape 2: Faire Y
  3. Étape 3: Faire Z

  Problèmes:
  1. Ce plan existe seulement dans l'historique conversationnel
  2. Après 50 messages, le plan est "loin" dans mon contexte
  3. Je n'ai aucun mécanisme de checkpoint - je ne peux pas marquer "Étape 2 DONE" de façon
   durable
  4. Si vous m'interrompez ou si je rencontre une erreur, je dois relire tout l'historique
  pour savoir où j'en suis
  5. Les étapes sont vagues ("Créer contenu ExamPrep") - pas de critère clair de "fini"

  POURQUOI API TASKS FONCTIONNE:

  # Je crée les tâches UNE FOIS
  curl POST /api/tasks → Tâche #260 créée

  # À tout moment, je peux vérifier
  curl GET /api/tasks/260 → {"status": "IN_PROGRESS"}

  # Quand je termine
  curl PATCH /api/tasks/260 {"status":"DONE"}

  # La prochaine fois
  curl GET /api/tasks?status=TODO → Tâche #267 suivante

  Avantages:
  1. État persistant - même si la conversation recommence, les tâches sont là
  2. Atomique - 1 tâche = 1 action vérifiable (ex: "Fix localisation")
  3. Séquentiel forcé - Je ne peux PAS passer à #267 sans finir #260
  4. Traçable - Historique complet des changements d'état

  ANALOGIE:

  Mode Plan = Faire les courses avec une liste mentale → J'oublie le lait
  API Tasks = Faire les courses avec une app (Todoist) → Je coche chaque item, impossible
  d'oublier
